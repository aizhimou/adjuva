# Adjuva

## A Persistent Project Assistant

---

# What is Adjuva?

Adjuva is a persistent project assistant designed for long-term human–AI collaboration.

Unlike traditional AI assistants that focus on answering questions or completing isolated tasks, Adjuva stays with your projects over time.

It observes.

It remembers.

It follows up.

It prepares drafts.

It reports progress.

It asks for approval when needed.

And it continues working when you're away.

Adjuva is not another coding IDE.

It is not another autonomous AI agent.

It is a trusted aide for your projects.

---

# Name Origin

## Adjuva

The name **Adjuva** is derived from the Latin root:

> adjuvare

Meaning:

> to help
> to assist
> to support
> to aid

The same root appears in several English words related to assistance and support.

Rather than emphasizing autonomy or replacement, Adjuva emphasizes collaboration.

The goal is not to replace the project owner.

The goal is to help them move forward.

---

# Why Adjuva?

Most AI products today focus on intelligence.

They try to answer:

> How can AI do more?

Adjuva focuses on a different question:

> How can humans and AI work together over time?

Great projects are rarely built through a single breakthrough.

They advance through:

* reminders
* decisions
* drafts
* reviews
* conversations
* follow-ups
* small actions repeated consistently

These activities are often tedious, repetitive, and easy to forget.

Yet they determine whether a project succeeds.

Adjuva exists to carry part of that burden.

---

# The Philosophy

## AI as an Aide, not a Replacement

Most agent systems are built around autonomy.

The ideal outcome is often described as:

> Give the AI a task and let it handle everything.

Adjuva takes a different approach.

Many important decisions:

* strategic choices
* public communication
* product direction
* personal priorities

should remain human decisions.

Instead of replacing the human, Adjuva acts as:

> a trusted aide

An aide does not take command.

An aide helps the commander make better decisions.

---

# Human ↔ Agent

Most modern AI tooling focuses on:

```text
Agent ↔ Code
```

Examples:

* Cursor
* Superset
* Claude Code
* Codex
* Gemini CLI

These systems help agents write software.

Adjuva focuses on:

```text
Human ↔ Agent
```

The goal is not to maximize code output.

The goal is to maximize long-term collaboration.

---

# Persistent Collaboration

Traditional AI interaction looks like this:

```text
Open Chat
↓
Ask Question
↓
Get Answer
↓
Close Chat
```

The relationship ends.

Adjuva introduces a different model:

```text
Project
↓
Agent observes
↓
Agent works
↓
Agent reports
↓
Human reviews
↓
Agent continues
```

The relationship persists.

The context persists.

The project persists.
